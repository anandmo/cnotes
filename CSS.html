CSS Structure 

selector { property :  value;}


Class selector - Can be used multiple times with html elements, grouping diffrent HTML elements into a class
Applying CSS properties on the class effect all the HTML elements under that class 

  <img class = "fox" src="images/image.jpg" alt="Fox Image">

.fox {
  width: 150px;
  height: 100px;
}


More than one class can be used with an html element, which is not possible with id 

.fox {
  width: 150px;
  height: 100px;
}

.circular {
  border-radius : 100;
}


 <img class = "fox circular" src="images/image.jpg" alt="Fox Image">


Id selector - used only once with a html element, rarely used 

 <img id = "fox" src="images/image.jpg" alt="Fox Image">

starts with # unlike class which starts with a . 

#fox {
  width: 150px;
  height: 100px;
}

The box model

All HTML elements are within a box

if in CSS property all the sides are mentioned in the following order
top right buttom left -> clock wise

The property of the box are
padding, border(default 3px), margin(sepration between current elemnt and any other elemnet), width and height


CSS has following Display property

block
inline
inline-block - width can be changed and only takes up as much space as is required to display the element
None - compeletly hide the elements


block element - take whole width of screen that is blocking any element to be on the same line ex : p, h1 to h6, div, list etc - by using css width can be changed

inline display element - only takes up as much space as is required to display the element ex: Spans (<span>...</span>), images (img) , anchors (<a>) - by using css width cannot be chnaged


display : none vs visibility : hidden
visibility hidden -> hide the elements by the space that the element required is still there, other elements cannot occupy that space

examples :

although p is block element it can be changed to inline by specifing in css
p {
display : inline;
}

similarly any other html element can have any other properties


CSS Positioning :

HTML are positioned by default
rules

  1. content is everthing
  2. order is deterimend by code
  3. children element sits over the parent element, i.e childeren will be closer to the viewer

if we don't want to go with the default layour rules we can use position property properties

position
static : keep to default html rules and flow
relative : doesn't effect the layout of any other HTML element

top,bottom,right,left -> moving the element does not displaces other elements
move element relative to where it was earlier
absolute : move element relative to its parent element, does effect the layout of other element
fixed : element position is fixed and does not changes even when scrolled

font size
16px = 100% = 1em

new feature in CSS called
rem - root em for font size

web-safe fonts : most browser should be able to support these fonts

body {
font-family : san-serif;
}

font-size : 90px; -> but this is not dynamic static size font

to make font size dynamic
font-size : 100%; which is equivalent to 16px font size;

another way for dynamic font size is to use em unit
font-size: 1em; which is equivalent to 16px

imp : em and % size are inherited which mean if we define default font size using % or em in parent html element ex : boby then the child element like h1 which add that size to its already existing size
to over come this use size in rem ex : 1rem , which basically tells browser to ignore parent element font size.

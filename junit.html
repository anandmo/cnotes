What is Junit ?
Junit is framework in Java to perform unit testing on Java Programs.
We can test indiviual functions by calling them and matching there results against the correct values and noting how much time it took to perform that task.

Junit 5 is the latest and has replaced Junit 4. Althought Junit 5 can run Junit 4 for backward compatiability.

Junit 5 is composed of the following

  1. Junit Vintage : To provide backward compatiability with Junit 4
  2. Junit jupiter : Gives all the new functionality
  3. Junit Platform : Act as an interface between Clint and Junit

Working of Junit

  1. Calls the function to be tested
  2. Output
  3. Matches Output with expected result
  4. if Result Matches test is success
  5. else Result does not Matches test fails

Junit 5 Assertions
Assertions are utility methods to support asserting conditions in tests
assertNull(), assertNotNull(), assertEquals(), assertNotEquals() , assertThrows(), assertTrue(), assertFalse(), assertSame(), assertNotSame(), Assertions.fail(), assertArrayEquals(), assertIterableEquals(), assertTimeout(), assertTimeoutPreemptively()

Example for Assertion
Junit 5 org.junit.jupiter.Assertions class provides different static assertions method to write test cases.
Please note that you need to use JUnit’s org.junit.Assert class in case of JUnit 4 or JUnit 3 to assert using assertNull method.

Assertions.assertNull() checks that object is null. In case, object is not null, it will through AssertError.

  • public static void assertNull(Object actual)
  • public static void assertNull(Object actual, String message)
  • public static void assertNull(Object actual, Supplier messageSupplier)

import java.util.HashMap;
import java.util.Map;
 
import org.junit.jupiter.api.Assertions;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
 
public class AssertNullTest {
 
	AssertNullTest ant;
	
    public String getCapital(String country){
        Map<String,String> countryCapitalMap = new HashMap<String,String>();
        countryCapitalMap.put("Italy", "Rome");
        countryCapitalMap.put("Japan", "Tokyo");
        countryCapitalMap.put("Zimbabwe", "Harare");
        countryCapitalMap.put("Belgiaum", "Brussels");
        return countryCapitalMap.get(country);
    }
     
    @BeforeEach
	public void beforeEachTest()
	{
    	ant = new AssertNullTest();
	}
    /*
     * Examples for each overloaded methods of assertNull
     */
    
    //public static void assertNull(Object actual)
    @Test
    public void testIndia(){
    	String capitalIndia=ant.getCapital("India");
    	Assertions.assertNull(capitalIndia);
    }
    
    //public static void assertNull(Object actual, String message)
    @Test
    public void testUSA(){
    	String capitalUSA=ant.getCapital("USA");
    	// You can pass message as well
    	Assertions.assertNull(capitalUSA,"Capital should be null");
    }
    
    //public static void assertNull(Object actual, Supplier messageSupplier)
    @Test
    public void testJapan(){
    	String capitalJapan=ant.getCapital("Japan");
    	Assertions.assertNull(capitalJapan,() -> "Capital should be null");
    }
}

testIndia – Pass As countryCapitalMap does not contain key “India”, assertNull will return true and this testcase will pass.
testUSA – Pass As countryCapitalMap does not contain key “USA”, assertNull will return true and this testcase will pass.
testJapan – Fail As countryCapitalMap contains key “Japan”, assertNull will return false and this testcase will fail.

Nested Test Cases
We can have multiple innner classes within one Test class. Each inner class can have there own Test methods /cases. The only thing to make sure here is that the inner class are annotated with @Nested, so that the compiler will inculde these cases as well.
Nested cases are helpful in grouping similar test cases together without creating an seperate class. 

class Test_Main {
    
	@BeforeAll
	static void beforeallfunction(){
    
	}
    
	@AfterAll
	static void afterallfunction(){

	}
    
	@BeforeEach
	void beforeeachfunction(){

        
	}
    
	@AfterEach
	void aftereachfunction(){

	}
    
	@Test
	void MyMaintest1(){
	//Test 1
	}
    
	@Test
	void MyMainTest2(){
	//Test 2	
	}
    
	@Nested
	class Test_Nested {
    
		@BeforeEach
		void nestedBeforeEach(){
        	
		}
        
		@AfterEach
		void nestedAfterEach(){
        	
		}
        
		@Test
		void MyNestedTest1(){
            
		}
        
		@Test
		void MyNestedTest1(){
            
		}
        
	}
    
}

Nested class can have both @BeforeEach and @AfterEach methods.
To set @BeforeAll and @AfterAll you need to set the type as per-class as by default its per method.

Repeated Test
A test case can be repeated x-number of times by using the using the annotation @RepeatedTest(x). This can be used to test the performance. 

class Test {

	@Test
	@RepeatedTest(120)
	public void testCase(){
		//Content Here
	}
    
}

We can use @RepeatedTest with Nested Class Test cases.

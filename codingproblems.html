Find the maximum sum of a continuous subsequence in an array of integers.

e.g:

input: 1,-3,2,5,-8

output: 7

_______________

SOLUTION: 


public static int maxSubSequence(int[] arr) {

	if (arr == null || arr.length == 0) {

		return Integer.MIN_VALUE;

	}



	int maxSoFar = arr[0], curMax = arr[0];

	for (int i = 1; i < arr.length; i++) {

		curMax = Math.max(curMax + arr[i], arr[i]);

		maxSoFar = Math.max(curMax, maxSoFar);

	}



	return maxSoFar;

}

RUNNING TIME COMPLEXITY:

O(N)

SPACE COMPLEXITY:

O(1)

Explanation

A more optimal solution to this problem is known as Kadane’s Algorithm. The concept works off of the simple idea that at each position in the array if we know what the previous maximum subarray sum is, we can deduce that the new max must be either the previous max or the previous max + the number in our current position. For example, if our previous max is 4 and our current number is -1, the new max remains 4. If the previous max is 4 but our current number is 2, then the max is reassigned to be 6. We track this as local_max = max(local_max, local_max + num).

This is an easy way of knowing at any point in time, what the current streak is that we are on, but it does not capture the greatest subarray. For that purpose, we have a second variable global_max. After each reassessment of local_max, we check if the newly assigned local_max is greater than our global_max. If it is, we reassign our global_max. This is tracked as global_max = max(local_max, global_max).

Once we iterate over the entire array, our output will be whatever global_max is set to. This algorithm produces a time complexity of O(n).


Recursion describes a situation where a function, once called, is called again before the first call completes and returns.
Recursive functions use something called “the call stack.”

2 main types of recursion
Direct recursion happens when a function calls it-self within it-self before the function returns or end.
Indirect recursion (or mutual recursion) occurs when a function calls another function, eventually resulting in the original function being called again.

polymorphism

Created Sunday 13 March 2022

Before going to polymorphism, let's review some OOPS fundamentals.
The 4 pillars of OOPS are Abstraction, Polymorphism, Inheritance, Encapsulation. I call them A-PIE.

  • In short, Abstraction means to hide the complex and only expose what is required. The inner workings are hidden. Example would be a Java Class.
  • Polymorphism means that an entity (methods and operator) can show multiple behaviors. The best example of Java for polymorphism is method overloading and method overriding.
  • The use case for Inheritance is code reuse, so that we don't have to reinvent the wheel every-time.
  • Simply put Encapsulation means to restrict the data access. This can be done in java using public, private, protected and default access modifiers.

There are certain best practice we can follow while working in OOPS

  • The principle is called DRY which means Don’t Repeat Yourself . Having two blocks of the same code in two different places is redundant and increases code complexity.
  • All class and methods should always have only one functionality (Single Responsibility)
  • All methods and classes should be closed for modification but open for an extension (Open Closed Design)
Let's go through the below program which is an example of polymorphism, where instance method overriding is happening and dynamic binding happens during runtime. 

class Animal {
  public void animalSound() {
  System.out.println("The animal makes a sound");
 }
}
 
class Pig extends Animal {
@Override
 public void animalSound() {
 System.out.println("The pig says: wee wee");
 }
}
class Dog extends Animal {
 @Override
 public void animalSound() {
 System.out.println("The dog says: bow wow");
 }
}
 
class Main {
 public static void main(String[] args) {
 Animal myAnimal = new Animal(); //object creation happens at runtime
 Animal myPig = new Pig(); //upcasting
 Animal myDog = new Dog(); //upcasting
 
//jvm only knows which animalSound() to call based on the object created
 at runtime pointed by the reference, hence dynamic binding
 myAnimal.animalSound();
 myPig.animalSound();
 myDog.animalSound();
 }
}

Output :
The animal makes a sound
The pig says: wee wee
The dog says: bow wow

The above program should clear up the difference between method overloading (static binding) and method overriding (dynamic binding)
So, Method overloading in Java happens when a class has more than one method of the same name and their parameters are different. And creating a method in a child class which is already present in the parent class is known as method overriding. In java the child class method is followed by @Override annotation.

At this time let us also understand one more term called Up-casting. When we create a child class instance and store it in parent class reference ,this is also known as up-casting or widening
Parent p = new Child();

In up-casting the reference can access the parent class members but it cannot access all the child class members. Instead reference p can access only the overridden methods.
Note : Child class is a subset of parent class. Dog is-a animal. A subclass can access all public methods of super-class.

Lets come back to Static vs Dynamic binding
Association of method definition to the method call is known as binding. There are two types of binding: Static binding and dynamic binding
Static binding happens during the compile time, when the code is compiled by the compiler. The compiler is well aware which class is calling the methods.

Note : Binding of static (belongs to the class not any instance), private (hidden from outside access) and final (cannot change) methods is compile-time or static. because these methods cannot be overridden and the class type is determined at the compile time.

This brings us to one more important concept of virtual functions in programming. In many programming languages like C++ a function in parent class can only be overridden in child class only if it is declared as virtual with the help of virtual keyword.

Note : In Java all the functions except static or private or final are virtual functions by default.

In contrast, in C++ all the class member functions are non-virtual by default. They are made virtual by using the virtual keyword in the function signature.if the function of base class is made virtual then the function of Derived or Child class’s function with the same name can override the Base class’s function.
Since in Java all methods are virtual by default which means any method can be overridden when used in inheritance, unless that method is declared as final or static.

This brings us to one more important concept called VIrtual Method Table.

As per Wikipedia :
“A virtual method table (VMT), virtual function table, virtual call table, dispatch table, vtable, or vftable is a mechanism used in a programming language to support dynamic dispatch (or run-time method binding).The table are used at runtime to invoke the appropriate function implementations, because at compile time it may not yet be known if the base function is to be called or a derived one implemented by a class that inherits from the base class.”

So, it is clear that in method overriding, the method is bound to the objects on the runtime (as object creation ex : Temp temp = new obj1 happens at runtime) which is decided by the JVM. That’s why it is called Run time polymorphism.

Lets summarize the difference static and dynamic binding :
Static binding is a type of polymorphism that collects the information to call a method during the compile-time, it is faster.
Dynamic binding is a type of polymorphism that collects the information to call a method at the runtime, it is comparatively slower.

Let's end this article by again quoting Wikipedia definition of polymorphism
“In programming languages and type theory, polymorphism is the provision of a single interface to entities of different types or the use of a single symbol to represent multiple different types. The concept is borrowed from a principle in biology where an organism or species can have many different forms or stages.”

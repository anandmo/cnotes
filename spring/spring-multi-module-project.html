spring-multi-module-project

Created Friday 25 March 2022

spliting the project in logical unit or modules for easy refactoring, reuse and development
one single maven command can build all modules and maven can take care of build order

UML Diagram for the project

create a multi-module maven project
create a sub project

parent-project (also called pom project) CONTAINS child-project (modules)

parent-project does not contain any code, since its just a project that specifies the modules
as a result we can delete the source directory in parent project since its not required
add <packaging>pom</packaging> in pom.xml of this parent-project
pom is a container of sub modules
each sub-project is represented as a directory in the parent-project

create sub-project/sub-module
in eclipse :
1. right click the parent project -> new project

2. select project type as maven -> maven module

3. enter the child project module name. Parent Module will be pre-selected


here core is the name of the child project, by default child project is jar 

  <modelVersion>4.0.0</modelVersion>
  
    <parent>
       <groupId>com.blogspot.nodemorph</groupId>
       <artifactId>guess-the-number-game</artifactId>
       <version>0.0.1-SNAPSHOT</version>
    </parent>
    
  <artifactId>core</artifactId>
  <name>core</name>

similarly parent project pom.xml is updated automatically to have the child module name 

<modelVersion>4.0.0</modelVersion>
  
  <groupId>com.blogspot.nodemorph</groupId>
  <artifactId>guess-the-number-game</artifactId>
  <version>0.0.1-SNAPSHOT</version>
  <name>guess-the-number-game</name>
  
  <packaging>pom</packaging>
  
  <modules>
  	<module>core</module>
  </modules>

adding dependences to maven multi-module project
dependency can be added in parent project, child project can then inherite the dependency 

<dependencyManagement>
		<dependencies>
			<dependency>
				<groupId>ch.qos.logback</groupId>
				<artifactId>logback-classic</artifactId>
				<version>1.2.3</version>
			</dependency>
		</dependencies>
</dependencyManagement>

maven properties can be used to specify the version number
maven properties are values place holder, there value are accessible from anywhere within POM

parent project dependencies 

	<dependencyManagement>
		<dependencies>

			<!-- logback -->
			<dependency>
				<groupId>ch.qos.logback</groupId>
				<artifactId>logback-classic</artifactId>
				<version>${logback.version}</version>
			</dependency>

			<!-- spring context -->
			<dependency>
				<groupId>org.springframework</groupId>
				<artifactId>spring-context</artifactId>
				<version>${spring-context.version}</version>
			</dependency>

		</dependencies>
	</dependencyManagement>

child project dependencies 

	<dependencies>

		<!-- logback -->
		<dependency>
			<groupId>ch.qos.logback</groupId>
			<artifactId>logback-classic</artifactId>
		</dependency>

		<!-- spring context -->
		<dependency>
			<groupId>org.springframework</groupId>
			<artifactId>spring-context</artifactId>
		</dependency>

	</dependencies>

as seen the child project dependency does not have the version for the dependencies, as the version by default inherited from the parent

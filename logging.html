logging

Created Thursday 24 March 2022

logging helps to see the behaviour of code both during development of the code and after the software has been shiped
using logging api we can filter the logs, which is not the case if we just print the statements on console

SLF4J - The Simple Logging Facade for Java (SLF4J) serves as a simple facade or abstraction for various logging frameworks (e.g. java.util.logging, logback, log4j) allowing the end user to plug in the desired logging framework at deployment time.
Note : It is not itself a logging library, but a generic interface to one of many logging libraries

So with SLF4J as a facade, we can use either log4j or logback or <any other compatible logging framework>.

logback - it is logging framework based on slf4j. has three main components : loggers (for capturing logs), appenders (sending or printing logs to various destination) and layout (for formating the logs)
logback uses filters
logging behaviour can be configured at run time using a configuration file
supports xml and groovy based configuration

to add logback to a maven project go to https://mvnrepository.com/
search for logback-classic module
choose version Logback Classic Module Â» 1.2.3
copy the dependency written in xml
add the below to pom.xml within dependencies tag 

<dependency>
    <groupId>ch.qos.logback</groupId>
    <artifactId>logback-classic</artifactId>
    <version>1.2.3</version>
</dependency>

note : slf4j dependency are also automatically added as these are transative dependency i.e. logback depends on slf4j
maven will automatically download the dependency on the local system within a .m2/ directory

java class using logback 

package com.blogspot.nodemorph;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class HelloMaven {
	private final static Logger log = LoggerFactory.getLogger(HelloMaven.class);

	public static void main(String[] args) {
		log.info("hello info");
		log.debug("hello debug");
	}
}

17:28:10.044 [main] INFO com.blogspot.nodemorph.HelloMaven - hello info
17:28:10.049 [main] DEBUG com.blogspot.nodemorph.HelloMaven - hello debug

The org.slf4j.Logger interface is the main user entry point of SLF4J API. It is expected that logging takes place through concrete implementations of this interface.
The LoggerFactory is a utility class producing Loggers for various logging APIs, most notably for log4j, logback and JDK 1.4 logging.
logback uses default minimal configuration if it is not configured

logback initilization steps : Intially tries to find logback.groovy files , if not found tries to find logback.xml , if it fails to find it fall back to default configuration

logback levels : TRACE, DEBUG, INFO, WARN and ERROR

% is the required prefix
-5 tells how many letters can be used, 5 is the max length of the strings INFO, DEBUG, - means its left justified, leaving the - will make it right justified
{40} tells the max length of the logger name, also - is just a literal and working as a seperator
message, tells the actual logger message that comes from the program
n, new line 

<configuration>
	<appender name="STDOUT"
		class="ch.qos.logback.core.ConsoleAppender">

		<encoder
			class="ch.qos.logback.classic.encoder.PatternLayoutEncoder">
			<pattern>
				%date [%thread] [%-5level] %logger{40} - %message%n
			</pattern>
		</encoder>
	</appender>

	<logger name="com.blogspot.nodemorph" level="DEBUG" />

	<root level="INFO">
		<appender-ref ref="STDOUT" />
	</root>
</configuration>

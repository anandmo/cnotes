a maven project has pom.xml file 

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.blogspot.nodemorph</groupId> <!-- group id -->
  <artifactId>hello-maven</artifactId>  <!-- artifact id -->
  <version>0.0.1-SNAPSHOT</version>
  <name>hello-maven</name> <!-- project name -->
  <description>a demo project to understand maven</description>
</project>

maven life cycle - There are three built-in build lifecycles: default, clean and site. The default lifecycle handles your project deployment, the clean lifecycle handles project cleaning, while the site lifecycle handles the creation of your project's web site.

Each lifecycle has different phases in them as given below. A phase is a stage/step in that lifecycle. Remember when a phase is executed, it executes the previous phases as well.

Ex. When “compile” phase is executed, the previous phases validate, Initialize, generate-sources, process-sources, generate-resources, process-resources are also executed.


Example Commands to execute maven phases:

mvn clean
mvn compile
mvn install
mvn site

A Lifecycle is Made Up of Phases. These lifecycle phases are executed sequentially to complete the lifecycle.

Goals are the individual task. In phases, maven runs the previous phases as well when called. But, goals run the particular task of compile, package, test, etc.
For ex, we have seen “compile” phase executes the previous phases validate, Initialize, generate-sources, process-sources, generate-resources, process-resources. But if we execute compile as a maven goal, it just executes that compile step alone.

Maven internally calls different plugins to do every task. When you call a phase or a goal, the relevant plugins are called to execute them.

When we call “mvn compiler:compile” goal, the “maven-compiler-plugin” alone gets called.

maven plugins - Plugins are artifacts that provide goals to Maven. Furthermore, a plugin may have one or more goals wherein each goal represents a capability of that plugin. For example, the Compiler plugin has two goals: compile and testCompile.

maven plugins and goal - A Build Phase is Made Up of Plugin Goals

Each Lifecycle has different phases. And each phases can have one to multiple goals.

https://maven.apache.org/guides/introduction/introduction-to-the-lifecycle.html

example :
mvn clean install

to fix jdk version used by maven we need to a new maven compiler plugin and define the newer java version there in pom.xml 

<project xmlns="http://maven.apache.org/POM/4.0.0"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
	<modelVersion>4.0.0</modelVersion>
	<groupId>com.blogspot.nodemorph</groupId>
	<artifactId>hello-maven</artifactId>
	<version>0.0.1-SNAPSHOT</version>
	<name>hello-maven</name> <!-- project name -->
	<description>a demo project to understand maven</description>

	<!-- setting up the maven compiler plugin to compile code in java 10 -->
	<build>
		<pluginManagement>
			<plugins>
				<plugin>
					<groupId>org.apache.maven.plugins</groupId>
					<artifactId>maven-compiler-plugin</artifactId>
					<version>3.7.0</version>
					<configuration>
						<target>10</target>
						<source>10</source>
						<release>10</release>
					</configuration>
				</plugin>
			</plugins>
		</pluginManagement>
	</build>

</project>
